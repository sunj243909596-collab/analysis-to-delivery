# 安全批量编辑模式

> 适用于跨多文档的批量修改
> 防止误改、误删、跨文档残留

## 危险操作清单

- ❌ `sed -i` 在大量文件上
- ❌ 跨文档 `patch` 操作
- ❌ 模糊匹配的 `replace_all`
- ❌ 不备份直接修改

## 安全模式

### 1. 备份优先

```bash
# 备份原文件
cp -r docs/ docs.backup-$(date +%Y%m%d)
```

### 2. 明确边界

- 修改前先 grep 确认影响范围
- 不要用通配符 `*` 修改 100+ 文件
- 拆分成小批次（每批 5-10 文件）

### 3. 验证清理

修改后必须：
```bash
# 确认删除内容已清除（应该 = 0）
grep -r "旧内容" docs/ | wc -l

# 确认新增内容已出现（应该 > 0）
grep -r "新内容" docs/ | wc -l

# 清理跨文档残留
grep -r "已删除的关键词" docs/  # 应该为空
```

### 4. 文档版本号

每个修改必须更新版本号和修订记录。

## 推荐工具

- `git diff` 查看修改
- `git log` 查看历史
- 文本编辑器多光标（VSCode / Cursor）
- 专用 diff 工具（Meld / Beyond Compare）
