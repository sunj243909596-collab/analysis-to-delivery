# QA 审计清单

> 适用于阶段 9（QA 审计）
> 6 大类全量审计

## 审计维度

### 1. 公式正确性

- 全文搜索旧公式残留
- 对比公式前后版本差异
- 业务含义是否一致

### 2. SQL 方言一致性

- 严禁 Oracle/PG 混用
- 跑 `scripts/sql-dialect-check.py` 自动检查
- 手动检查 rownum/limit、nvl/coalesce、sysdate/now 等

### 3. 部署清单完整性

- 新增表都在部署清单里？
- 索引都列了？
- 初始数据脚本完整？

### 4. 跨文档交叉引用一致性

- BRD ↔ FSD ↔ PRD ↔ 数据模型 ↔ 开发设计
- 同一字段名/类型/长度/可空性必须一致
- 业务规则描述必须一致

### 5. 字段映射一致性

- 跑 `scripts/field-alignment-check.py`
- 文档中引用的字段是否与知识库一致
- 业务同义不同名是否显式标注

### 6. 删除/修改的精度

- 逻辑层（WHERE/CASE/JOIN）删除是否彻底？
- 数据层（SELECT 输出）删除是否保留？
- 跨文档残留是否清理？

## 审计流程

```bash
# 1. 跑全量审计
python3 scripts/full-qa-audit.py {项目目录}

# 2. 人工阅读审计报告

# 3. 按 P0/P1/P2 分级
#    P0: 必须修复（严重缺陷、权限缺失、审计缺失）
#    P1: 建议修复（主要缺陷、追溯不全）
#    P2: 优化建议（一般缺陷、功能优化）

# 4. 委派 Claude 修复 P0

# 5. 重跑确认 P0 修复
```

## 修复迭代规则

1. 首次失败：读取错误输出 → 定位问题文件/行号 → 修复 → 重新验证
2. 二次失败：扩大搜索范围，检查关联文件 → 修复 → 重新验证
3. 三次失败：停止自动修复，输出完整错误报告 + 修复建议，等用户决策
