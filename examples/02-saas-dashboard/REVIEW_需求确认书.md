# REVIEW 需求确认书:客户订单管理

> 项目:SaaS 后台 — 客户订单管理
> 日期:2026-06-22
> 状态:✅ AI 理解已确认(待用户最终签字)

## 一、需求背景确认（AI 助手的理解）

### 1.1 业务目标理解

| 用户原话 | AI 理解 | 置信度 |
|---|---|---|
| "全流程在线化" | 订单从创建到完成全在 SaaS 后台内完成,无 Excel / 微信群流程 | 95% |
| "提升效率 50%" | 30 分钟单 → 15 分钟单(参考 WMS 示例锚点) | 90% |
| "错误率 < 0.5%" | 表单校验 + 双重确认(下单+支付) | 90% |

### 1.2 角色理解

5 角色 + 多租户隔离,与 WMS 4 角色不同(无质检员角色,改为退款审核归销售)。

### 1.3 状态机理解

8 个状态:`draft` / `submitted` / `paid` / `shipped` / `completed` / `cancelled` / `refunded` / `partial_refunded`(部分退款)。

## 二、需求目标确认（待确认的设计假设）

| 假设 | 默认值 | 影响 | 需确认 |
|---|---|---|---|
| 支付通道 | 支付宝/微信/Stripe 三选一 | 集成工作量 | ⚠️ |
| 物流对接 | 顺丰 API | 实时物流跟踪 | ⚠️ |
| 多租户隔离 | 行级 `tenant_id` | 简单但要求所有查询带条件 | ✅ |
| 数据看板 | T+1 离线数仓 | 不要求实时 | ✅ |
| 国际化 | 中文 only(v1) | 不需要 i18n | ✅ |
| 移动端 | 响应式 Web(v1) | 不需要 APP / 小程序 | ✅ |
| 货币 | CNY(v1) | 单币种 | ✅ |

## 三、功能范围确认（与 WMS 示例的差异说明）

| 维度 | 假设差异 | 处理方式 |
|---|---|---|
| 数据库 | Oracle → PostgreSQL | 改用 `COALESCE` / `LIMIT` / `SERIAL` / `BIGSERIAL` / `OFFSET` |
| 状态码 | 2 位数字 → 字符串枚举 | 改用 TypeScript enum + 字符串存储 |
| 字符分隔 | `\|\|` 通用 | PG 也支持 `\|\|`,无需改 |
| 整数除法 | `1.0/3` 必须 | PG 也需要 |
| 审计字段 | 五件套必须 | 一致 |

## 四、合规要求确认（字段映射预测,待阶段 1.3 字段对齐）

| 业务概念 | 字段(预测) | 备注 |
|---|---|---|
| 订单号 | `orders.order_no`(VARCHAR(20)) | UUID 简化版 |
| 客户 ID | `orders.tenant_id` + `orders.customer_id` | 多租户 |
| 订单状态 | `orders.status`(VARCHAR(20)) | 字符串枚举 |
| 创建时间 | `orders.created_at`(TIMESTAMPTZ) | PG 专用类型 |
| 乐观锁 | `orders.version`(INTEGER) | 与 WMOS `WM_VERSION_ID` 类似 |
| 金额 | `order_items.unit_price`(NUMERIC(10,2)) | PG NUMERIC 防浮点 |
