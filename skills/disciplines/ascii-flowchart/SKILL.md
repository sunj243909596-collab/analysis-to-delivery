# ASCII-Flowchart — ASCII 流程图规范

## 为什么不用 Mermaid

- 用户验证:终端渲染差(`"还是改为原来的吧,更不清楚了"`)
- ASCII 在任何终端/编辑器都能正确显示
- 简单、稳定、可 grep

## 工具选择

| 类型 | 工具 |
|---|---|
| 状态流转图 | **ASCII 文本字符**(强制) |
| 业务流程图 | ASCII / Draw.io(简单用 ASCII,复杂用 Draw.io) |
| 泳道图 | ASCII(角色分工清晰) |

## 画法规范

### 1. 角色与流程节点

```
角色A              系统              角色B
  │                │                  │
  │ ① 动作1        │                  │
  ├──────────────→│                  │
  │                │ ② 内部处理       │
  │                ├──────────────→│
  │                │                  │ ③ 动作3
  │ ④ 结果返回     │                  │
  │←──────────────┤                  │
```

### 2. 回流闭环(强制)

所有回流路径箭头**必须汇聚到同一目标节点并画线连回**。

```
        ┌──────┐
        ▼      │
    ┌──────┐  │
    │ 状态A │──┘
    └──────┘
```

**反例**:回流箭头指向中间,造成视觉混乱。

### 3. 异常分支

```
│ 异常分支:        │
│ ⑩ 短收/拒收/超收  │
├─────────────────→│
│                   │ 推送给质检员
```

## 字符集

```
─ │ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼ ← → ↑ ↓
※ ① ② ③ ④ ⑤ ⑥ ⑦ ⑧ ⑨ ⑩
✅ ⚠️ ❌ 🔄 ❓
```

## 反例

```markdown
❌ 错误:用 Mermaid
\`\`\`mermaid
graph LR
    A --> B
\`\`\`

✅ 正确:用 ASCII
A → B
```
