# 需求确认表：客户订单管理

> 生成时间：2026-06-22
> 状态：✅ 已确认
> 适用版本：analysis-to-delivery v3.0.0-dev

## 一、需求背景

- **目标客户**：B2B 中小企业（零售/批发）
- **核心痛点**：当前用 Excel + 微信群管理订单，易出错、难追溯
- **目标**：全流程在线化，提升效率 50%，错误率 < 0.5%

| 角色 | 场景 | 权限 |
|---|---|---|
| 客户（下单人） | 下单/查询/支付/申请退款 | 看自己的订单 |
| 销售（跟单员） | 审核订单/确认收款/安排发货 | 看所辖客户订单 |
| 仓管 | 拣货/发货/更新物流 | 改订单状态 |
| 管理员 | 客户管理/权限分配 | 全部 |
| 数据看板 | 看销售统计 | 只读 |

## 二、需求目标

- 客户注册/登录（多租户）
- 商品目录（SKU 管理）
- 订单创建（草稿→提交→支付）
- 支付集成（支付宝/微信/Stripe）
- 发货/物流跟踪
- 退款流程
- 数据看板（销售统计）
- 客户隔离（每个客户只看自己的订单）

成功指标：

- 订单处理效率提升 50%
- 错误率 < 0.5%
- 验收标准：草稿到完成全流程 < 24h

## 三、功能范围

**包含**：

- 客户管理（注册/登录/多租户隔离）
- 商品目录（SKU）
- 订单生命周期：
  ```
  draft → submitted → paid → shipped → completed
                  ↘ cancelled
                  ↘ refunded (从 paid/shipped 退)
  ```
- 数据集合：`customers`、`products`、`orders`、`order_items`、`payments`、`shipments`、`refunds`、`users`

**排除**：

- 财务对账（独立模块）
- 营销活动（独立模块）
