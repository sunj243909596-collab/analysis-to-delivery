# 需求确认表：会员积分管理

> 生成时间：2026-06-22
> 状态：✅ 已确认
> 适用版本：analysis-to-delivery v4.0.0

## 一、需求背景

- **目标客户**：B2C 零售品牌会员（C 端用户）
- **核心痛点**：线下门店用纸质会员卡 + 微信群发积分，易丢失，用户感知弱
- **目标**：积分全流程在线化，提升复购率 30%，激活沉睡用户

| 角色 | 场景 | 权限 |
|---|---|---|
| 游客 | 浏览首页 / 注册 | 无 |
| 会员 | 查看积分 / 兑换 / 签到 / 邀请 | 看自己的积分 |
| VIP 会员 | 同上 + 专属兑换 | 看自己的积分 |
| 店员 | 扫码核销 | 改订单状态 |
| 运营 | 配置活动 / 调整积分 | 改积分规则 |
| 管理员 | 用户管理 / 权限 | 全部 |

## 二、需求目标

- 注册/登录（手机号 + 微信 / Apple ID）
- 签到领积分（每日 / 连续）
- 消费返积分（扫码绑定小票）
- 邀请好友得积分
- 积分兑换商品 / 优惠券
- 积分过期提醒（到期前 7 天推送）
- 积分明细（收入 / 支出 / 过期）
- 会员等级（普通 / 银 / 金 / 钻石）
- 推送通知（FCM）
- 数据分析（运营后台）

成功指标：

- 签到响应 < 500ms
- 兑换流程 < 3 步
- 积分过期提醒准确率 100%
- 离线签到成功率 > 95%
- 推送送达率 > 90%
- App 启动时间 < 2s
- Crash 率 < 0.1%

## 三、功能范围

**包含**：

- 积分生命周期：
  ```
  earn(获取)→ active(生效中,未到期)→ used(已使用)
                       ↓
                    expired(过期)
                       ↓
                 frozen(冻结,争议中)
  ```
- 兑换订单状态：
  ```
  draft(草稿)→ submitted(已提交)→ approved(已审核)→ fulfilled(已核销)→ completed(已完成)
                  ↘ rejected(已拒绝)
                  ↘ cancelled(已取消)
  ```
- 数据集合（Firestore）：`users`、`points_wallets`、`points_ledger`、`points_batches`、`redemptions`、`products`、`coupons`、`referrals`、`check_ins`、`notifications`

**排除**：

- 商城主链路（独立模块）
- 支付通道（复用主 App 支付）
