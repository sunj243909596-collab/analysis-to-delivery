## 一、项目概述

### 1.1 业务背景

B2C 零售品牌当前用纸质会员卡 + 微信群发积分,存在以下问题:
- 易丢失:会员卡丢失,积分无法追溯
- 感知弱:用户不知道有多少积分、即将过期
- 兑换难:兑换流程需到店,无在线兑换
- 复购低:沉睡用户激活成本高
- 数据散:积分数据在小票 / 微信 / 系统中分散

### 1.2 项目目标

- ✅ 积分全流程在线化(App 内完成所有操作)
- ✅ 复购率提升 30%
- ✅ 沉睡用户激活率 20%
- ✅ 积分过期主动提醒(到期前 7 天 Push)
- ✅ 离线签到成功率 > 95%
- ✅ 推送送达率 > 90%

### 1.3 项目范围

**包含**:
- 用户体系(注册 / 登录 / 会员等级)
- 积分获取(签到 / 消费返 / 邀请 / 活动)
- 积分兑换(商品 / 优惠券)
- 积分明细(收入 / 支出 / 过期)
- 积分过期(FIFO 批次消耗 + 过期提醒)
- 推送通知(FCM)
- 离线支持(签到 / 查看明细)
- 店员核销端(扫码)

**不包含**:
- 支付集成(v1 不涉及支付,纯积分兑换)
- 物流配送(兑换券由门店核销)
- 客服 IM(用第三方 SDK,如腾讯云 IM)
- 数据分析后台(用 Firebase Analytics + BigQuery)

## 四、功能模块

### 4.1 用户体系
- 手机号验证码注册
- 微信 / Apple ID 第三方登录
- 用户档案(昵称 / 头像 / 等级)
- 会员等级(普通 / 银卡 / 金卡 / 钻石)
- 升级规则(累计积分门槛)

### 4.2 积分获取
- **每日签到**:1-10 分(连续签到奖励)
- **消费返积分**:1 元 = 1 分(扫码绑定小票)
- **邀请好友**:双向奖励(邀请人 100,被邀请 50)
- **活动奖励**:运营后台配置

### 4.3 积分兑换
- 商品兑换(实物,需物流)
- 优惠券兑换(电子券,立即发放)
- 会员专享(不同等级不同商品)
- 兑换流程(draft → submitted → approved → fulfilled → completed)

### 4.4 积分明细
- 收入流水(来源 / 时间 / 金额)
- 支出流水(兑换单 / 时间 / 金额)
- 过期流水(批次 / 过期时间)
- 冻结流水(争议中)

### 4.5 积分过期
- FIFO 批次消耗(先过期先用)
- 过期时间(默认 2 年)
- 到期前 7 天推送
- 已过期积分不可恢复

### 4.6 推送通知
- FCM 推送(iOS + Android)
- 推送场景(签到提醒 / 积分到账 / 兑换成功 / 过期提醒 / 活动)

### 4.7 离线支持
- 签到离线缓存(网络恢复后同步)
- 明细离线查看
- 兑换需联网

### 4.8 店员核销端
- 扫码核销(PDA / 手机)
- 核销记录查询
- 异常订单处理

## 六、非功能需求

### 6.1 性能
- 签到响应 < 500ms
- 积分查询 < 300ms(Firestore 监听)
- 兑换流程 < 3s
- App 启动 < 2s

### 6.2 可靠性
- 离线签到成功率 > 95%
- 积分流水不可丢失(append-only)
- 兑换订单状态严格流转
- 过期处理每日定时(02:00)

### 6.3 实时性
- 积分变动实时同步(Firestore listener)
- 兑换状态实时推送
- 离线 → 在线自动同步

### 6.4 推送
- FCM 送达率 > 90%
- 过期提醒提前 7 天
- 用户可关闭非关键推送

### 6.5 安全
- Firestore 安全规则(规则级权限控制)
- Cloud Functions Admin SDK
- 手机号加密存储
- 敏感操作日志

## 七、风险与约束

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 积分并发扣减 | 超扣 / 数据不一致 | Cloud Functions 事务 + 乐观锁 |
| FIFO 过期计算错误 | 用户投诉 | 单元测试覆盖 + 每日对账 |
| 离线签到冲突 | 重复签到 | 客户端去重 + 服务端幂等 |
| FCM 送达率低 | 提醒失效 | 多通道兜底(APNs / 极光)|
| Firestore 跨文档事务限制 | 复杂业务难实现 | 业务侧合并 + 单文档原子 |
| 积分通胀 | 兑换力下降 | 运营后台可调控汇率 |
