# 合规路径(SaaS 后台 — 客户订单管理)

> 项目:SaaS 后台 — 客户订单管理
> 路径配置:本文档被 `/setup-analysis-delivery` 阶段 1 引用
> 用途:明确本项目的合规要求与法规依据

## 一、合规等级

**等级:无(NONE)**

本项目为 B2B 商业 SaaS,非医药 / 金融 / 政企 / 医疗场景,**不涉及 GSP / 等保 / PCI-DSS / HIPAA / GDPR(主市场国内)** 等强制法规。

**依据**:
- 客户类型:B2B 中小企业(零售 / 批发)
- 数据类型:订单 / 支付 / 物流(非敏感个人健康信息)
- 市场范围:中国大陆(v1)
- 不处理:未成年人数据 / 特殊类别个人信息

## 二、遵循的"软合规"

虽然无强制法规,本项目仍遵循以下通用安全与合规规范:

### 2.1 数据安全

- ✅ HTTPS 强制(生产环境)
- ✅ 密码 bcrypt 加密(cost ≥ 12)
- ✅ JWT 短期(15min)+ 长期 Refresh(7d,可撤销)
- ✅ 敏感字段加密存储(身份证 / 银行卡号用 AES-256)
- ✅ SQL 注入防护(Prisma 参数化查询)
- ✅ XSS 防护(React 默认转义 + CSP 头)
- ✅ CSRF 防护(双 token 机制)
- ✅ 限流(防暴力破解 / 恶意爬取)

### 2.2 数据隔离

- ✅ 多租户行级隔离(`tenant_id` 必带)
- ✅ 应用层中间件强制 + 数据库层 CHECK 约束
- ✅ 跨租户访问需越权日志告警
- ✅ 定期跨租户泄露扫描脚本

### 2.3 审计

- ✅ 审计字段五件套(`created_at` / `created_by` / `updated_at` / `updated_by` / `version`)
- ✅ 关键操作日志(订单状态变更 / 退款 / 权限变更 / 客户管理)
- ✅ 软删除(`is_deleted`)而非物理删除
- ✅ 日志留存 ≥ 180 天(参考等保要求)

### 2.4 可用性

- ✅ 支付回调幂等性(`transaction_id` UNIQUE)
- ✅ 物流 API 异步队列 + 重试(指数退避)
- ✅ 数据库连接池监控(最大 100,预警 80)
- ✅ 关键接口熔断降级(支付 / 物流)
- ✅ 健康检查端点(`/healthz` / `/readyz`)

### 2.5 数据备份

- ✅ 数据库每日全量备份(异地)
- ✅ WAL 归档(15 分钟一次)
- ✅ 备份恢复演练(季度)
- ✅ RPO ≤ 15 分钟,RTO ≤ 1 小时

## 三、不涉及但可借鉴的规范

| 规范 | 借鉴点 | 原因 |
|---|---|---|
| GSP 医药追溯 | 审计字段五件套 / 状态机严格 | 商业通用最佳实践 |
| 等保 2.0 三级 | 数据备份 / 日志留存 6 个月 | 商业 SaaS 通用 |
| PCI-DSS | 支付卡号不落库 | 走第三方支付通道 |
| GDPR | 数据导出 / 删除 API | 海外客户可能要求 |
| SOC 2 | 访问控制 / 监控告警 | 海外客户准入要求 |
| ISO 27001 | 信息安全管理体系 | 大客户准入要求 |

## 四、未来合规扩展

如未来需要,以下扩展可平滑接入:

### 4.1 等保 2.0 三级(国内政企)

- 增加日志集中管理(ELK / Loki)
- 启用数据库审计插件(`pgaudit`)
- 完善备份恢复演练(季度)
- 双因素认证(2FA)
- 漏洞扫描(每月)
- 渗透测试(年度)

### 4.2 GDPR(海外客户)

- 增加 `data_subject_rights` API(导出 / 删除)
- Cookie 同意管理
- 数据驻留地域选择(欧盟 / 美国 / 亚太)
- 隐私政策生成器
- DPO 角色定义

### 4.3 PCI-DSS(若需自建支付)

- 网络分段(支付网段隔离)
- 加密传输 + 加密存储
- 严格的访问控制
- 定期 ASV 扫描
- 卡号 TOKEN 化

## 六、合规路径选择

- **不选择 GSP** — 非医药,无强制要求
- **不选择等保** — v1 客户无政企,v2 视情况
- **不选择 HIPAA** — 非医疗数据
- **不选择 PCI-DSS** — 走第三方支付,卡号不落库
- **不选择 GDPR** — v1 主市场国内
- **不选择 SOC 2** — v1 无海外客户

## 七、与 WMS 示例(01-wms-warehouse)的合规对比

| 维度 | WMS(01)| SaaS(02)| 差异 |
|---|---|---|---|
| 合规等级 | GSP(医药)| None | **核心差异** |
| 法规依据 | 《药品经营质量管理规范》| 无 | WMS 强约束 |
| 审计字段 | 五件套(必须)| 五件套(最佳实践)| WMS 强制 |
| 状态机校验 | 严格(15+ 状态)| 严格(9 状态)| 都严格 |
| 批次追溯 | 必备(GS001/GS002)| 不涉及 | WMS 核心 |
| 数据留存 | ≥ 5 年(GSP)| ≥ 180 天(参考)| WMS 更长 |
| 权限分级 | 4 级(强)| 5 角色(业务)| 都精细 |
| 越权告警 | 必须 | 建议 | WMS 强 |
| 验收测试 | UAT 必跑(法规)| 建议 UAT | 都建议 |
