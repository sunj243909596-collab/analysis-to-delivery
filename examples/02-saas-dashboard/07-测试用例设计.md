# 测试用例设计：SaaS 后台 — 客户订单管理

> 版本：v0.1
> 日期：2026-07-10
> 状态：✅ 已确认

## 一、测试范围

本测试用例覆盖 SaaS 后台客户订单管理的全部功能,包括：
- 客户下单主流程
- 支付集成（支付宝/微信/Stripe）
- 物流跟踪
- 退款流程
- 多租户数据隔离
- GDPR/PIPL 合规校验
- 性能与安全

测试范围以 `01-业务需求文档 BRD.md` §四 功能模块为准,涵盖 4.1-4.4 共 4 个核心模块 + 多租户 + 合规 6 大维度。

## 二、测试策略

| 测试类型 | 范围 | 工具 |
|---|---|---|
| 单元测试 | Service / Repository 层(订单/支付/退款核心业务) | Vitest + Jest |
| 集成测试 | Controller → Service → DB + Prisma 全链路 + 支付 mock | Supertest + Testcontainers (PG) |
| E2E 测试 | Web → 后端 → 支付 webhook → 物流 API 全链路 | Playwright + Cypress |
| 性能测试 | 100 单/秒并发 + 创建响应 < 500ms | k6 + Artillery |
| 安全测试 | 多租户隔离 + SQL 注入 + XSS + CSRF + 密码强度 | OWASP ZAP + 手工 |
