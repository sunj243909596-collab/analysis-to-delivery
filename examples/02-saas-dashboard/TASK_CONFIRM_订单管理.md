# TASK_CONFIRM:客户订单管理

> 项目:SaaS 后台 — 客户订单管理模块
> 日期:2026-06-22
> 状态:⬜ 待用户填写

## 1. 业务背景

- **目标客户**:B2B 中小企业(零售/批发)
- **核心痛点**:当前用 Excel + 微信群管理订单,易出错、难追溯
- **目标**:全流程在线化,提升效率 50%,错误率 < 0.5%

## 2. 角色与场景

| 角色 | 场景 | 权限 |
|---|---|---|
| 客户(下单人) | 下单/查询/支付/申请退款 | 看自己的订单 |
| 销售(跟单员) | 审核订单/确认收款/安排发货 | 看所辖客户订单 |
| 仓管 | 拣货/发货/更新物流 | 改订单状态 |
| 管理员 | 客户管理/权限分配 | 全部 |
| 数据看板 | 看销售统计 | 只读 |

## 3. 核心功能

- [ ] 客户注册/登录(多租户)
- [ ] 商品目录(SKU 管理)
- [ ] 订单创建(草稿→提交→支付)
- [ ] 支付集成(支付宝/微信/Stripe)
- [ ] 发货/物流跟踪
- [ ] 退款流程
- [ ] 数据看板(销售统计)
- [ ] 客户隔离(每个客户只看自己的订单)

## 4. 状态机

```
draft → submitted → paid → shipped → completed
                  ↘ cancelled
                  ↘ refunded (从 paid/shipped 退)
```
