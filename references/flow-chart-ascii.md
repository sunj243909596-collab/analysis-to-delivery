# ASCII 流程图规范

> 用户明确拒绝 Mermaid（"还是改为原来的吧，更不清楚了"）
> 状态流转图必须用 ASCII 文本字符

## 为什么不推荐 Mermaid

- ❌ 终端渲染差（用户验证）
- ❌ 信息密度低
- ❌ 布局混乱（特别是复杂状态机）
- ✅ ASCII 紧凑、一目了然、终端渲染稳定

## 状态流转图标准画法

### 基本元素

```
   ┌──────────┐
   │  节点名  │
   └─────┬────┘
         │  边标签（可选）
         ▼
   ┌──────────┐
   │  下个节点│
   └──────────┘
```

### 回流闭环（强制）

**所有回流路径的箭头必须汇聚到同一目标节点，并明确画线连回。**

```diff
   ┌──────────┐
   │ 30 收货中│
   └─────┬────┘
         │
         ├──→ 异常 → 处理 → 回到 30 ←────┐
         │                               │
         │  ◄────────────────────────────┘
         │
         ▼
   ┌──────────┐
   │ 40 收货完│
   └──────────┘
```

## 业务流程图（泳道图）

### 角色分列

```
仓管员          系统           复核员
  │              │              │
  │ ① 扫 ASN    │              │
  ├────────────→│              │
  │              │ ② 加载详情   │
  │ ③ 显示清单   │              │
  │←────────────┤              │
  │ ④ 扫 SKU   │              │
  ├────────────→│              │
  │              │ ⑤ 校验      │
  │              ├─────────────→│ ⑥ 审核
  │              │              │
```

## 状态码规范

每个状态节点必须包含：
- 状态码（数字）
- 状态名（中文）
- 可选：英文名

```
   ┌──────────────┐
   │ 10 新建/Open  │
   └──────────────┘
```

## 关键纪律

- ✅ 用等宽字体渲染（Courier New / Consolas / Monaco）
- ✅ 节点用 `┌─┐│└┘` 边框
- ✅ 箭头用 `─→` `▼` `▲`
- ✅ 回流用 `◄──` `←──┐`
- ❌ 不要用 Mermaid
- ❌ 不要用 Unicode emoji 装饰（用方括号 [GSP条款] 代替）

## 工具

如果 ASCII 在某终端显示错乱：
- 切换到等宽字体
- 或转用 `业务流程图.drawio`（详见 [drawio-guide.md](drawio-guide.md)）
