# 业务需求文档（BRD）：入库收货管理

> 项目：医药物流 WMS 收货管理优化
> 版本：v1.0
> 日期：2026-06-22
> 状态：✅ 已确认（待阶段 3-10 展开）

## 一、项目概述

### 1.1 业务背景
医药仓库到货收货环节，当前依赖纸质单据 + 手工录入，存在以下问题：
- 效率低：单 ASN 收货平均耗时 30 分钟
- 易出错：手工录入错误率约 2%
- 难追溯：无法满足 GSP 00201 条款的批次+序列号全流程追溯要求

### 1.2 项目目标
- ✅ 收货流程全电子化（巴枪扫码 + PC 审核）
- ✅ 收货效率提升 50%（30 分钟 → 15 分钟）
- ✅ 错误率降低至 0.1% 以下
- ✅ GSP 00201 追溯 100% 符合

### 1.3 项目范围
**包含**：
- ASN 单收货
- 扫码验证
- 上架
- PIX 记账
- 异常处理

**不包含**：
- ASN 预到货（独立模块）
- 上架策略配置（用 WMOS 原生）

## 二、角色与职责

| 角色 | 职责 | 权限 |
|---|---|---|
| 仓管员 | 巴枪扫码收货 | 创建收货单、扫码、提交 |

### 3.3 异常处理流程

| 异常类型 | 触发条件 | 处理方式 | 责任人 |
|---|---|---|---|
| 短收 | 实收 < 应收 | 提交短收单，待补货 | 仓管员 → 复核员 |
| 拒收 | 效期过期/质量不合格 | 提交拒收单，触发退货 | 仓管员 → 质检员 |
| 超收 | 实收 > 应收 | 系统拦截，不允许超收 | - |
| 序列号异常 | 序列号已存在 | 系统拦截 + 提示 | 仓管员 |
| 批次错误 | 批次与 ASN 不符 | 系统拦截 + 提示 | 仓管员 |

## 四、功能模块

### 4.1 巴枪端

#### 4.1.1 入库收货
- **入口**：巴枪主界面 → 入库收货
- **操作流程**：
  1. 扫 ASN 单号（TC_ASN_ID）
  2. 系统展示待收 SKU 清单
  3. 依次扫 SKU 条码 + 批号 + 序列号
  4. 系统实时校验（效期/质量/批号）
  5. 提交收货
- **关键校验**：
  - 效期过期 → 拒收
  - 质量不合格 → 拒收
  - 序列号重复 → 拦截
  - 批次与 ASN 不符 → 拦截

#### 4.1.2 收货异常
- **入口**：巴枪主界面 → 收货异常
- **功能**：
  - 短收确认
  - 拒收登记
  - 异常上报

### 4.2 PC 端

#### 4.2.1 收货管理
- **入口**：菜单 → 入库管理 → 收货管理
- **页面**：
  - 列表页：ASN 列表（搜索 + 筛选 + 分页）
  - 详情页：ASN 详情 + 收货明细
  - 审核页：复核员审核收货单
- **关键操作**：
  - 创建收货单
  - 提交审核
  - 审核通过/驳回
  - 异常处理

#### 4.2.2 收货记录查询
- **入口**：菜单 → 入库管理 → 收货记录
- **功能**：
  - 按 ASN/单号/日期/供应商查询
  - 查看收货详情
  - 导出 Excel

#### 4.2.3 收货统计
- **入口**：菜单 → 入库管理 → 收货统计
- **功能**：
  - 日/周/月收货量统计
  - 按供应商统计
  - 按 SKU 统计
  - 趋势图

## 五、数据要求

### 5.1 涉及的 WMOS 原生表

详见 `REVIEW_字段对齐分析.md`

主要表：
- `ASN`（ASN 单主表）
- `ASN_DETAIL`（ASN 明细表）
- `LPN`（货箱表）
- `INVENTORY`（库存表）

### 5.2 新增巴枪业务表

| 表名 | 用途 | 备注 |
|---|---|---|
| `APP_RCV_TASK` | 收货任务表 | 巴枪待办 |
| `APP_RCV_RECORD` | 收货记录表 | 每次扫码记录 |
| `APP_RCV_EXCEPTION` | 收货异常表 | 短收/拒收登记 |

### 5.3 关键字段映射

| 业务概念 | 字段 | 备注 |
|---|---|---|
| ASN 单号 | `ASN.TC_ASN_ID` | ❌ 严禁用 ASN_NBR |
| 货箱号 | `LPN.TC_LPN_ID` | ❌ 严禁用 LPN_NBR |
| 批号 | `ASN_DETAIL.BATCH_NBR` | ❌ 严禁用 LOT_ID |
| ASN 明细关联 | `ASN_DETAIL.ASN_DETAIL_ID` | ❌ 严禁用 ASN_ID |
| 订单组号 | `ASN.FTSR_NBR` | 业务上同义于"货主订单号" |

| 条款 | 缺陷等级 | 本项目设计 |
|---|---|---|

## 七、非功能需求

### 7.1 性能
- 单次扫码响应 < 1s
- 并发收货能力：50 单/分钟

### 7.2 可靠性
- 支持离线模式（断网仍能扫码，本地存储，网络恢复后同步）
- 数据一致性：所有 PIX 记账必须有事务保证

### 7.3 安全
- 巴枪设备绑定（防止越权扫码）
- 操作日志完整记录
- 敏感数据脱敏（如供应商联系人手机号显示 138****1234）

### 7.4 可维护性
- 严格分层（Controller / Service / Repository / Entity）
- 代码注释完整
- 单元测试覆盖率 ≥ 80%

## 八、风险与约束

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| PKMS 接口不稳定 | 质量状态查询失败 | 设计降级策略（允许手工确认质量状态） |
| WMOS PIX 接口变更 | 记账失败 | 严格按 WMOS PIX Matrix 实现，加版本兼容层 |
| 巴枪设备故障 | 收货中断 | 备用设备 + 数据本地缓存 |
| 上线时间紧 | 部分功能可能延期 | 按 P0/P1/P2 分级，P0 必须先上线 |
