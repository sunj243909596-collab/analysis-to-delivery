# 需求确认表：入库收货管理

> 生成时间：2026-06-22
> 状态：✅ 已确认
> 适用版本：analysis-to-delivery v1.0-mvp

## 一、需求背景

| 问题 | 你的回答 |
|------|---------|
| 业务痛点是什么？当前流程存在什么问题？ | 当前收货流程依赖纸质单据 + 手工录入，效率低、易出错、无法满足 GSP 00201 追溯要求 |
| 使用场景是什么？ | 医药仓库到货收货环节，仓管员用巴枪扫码 + 复核员 PC 端审核 |
| 涉及哪些角色？ | 仓管员、复核员、质检员、系统管理员 |
| 是否有现有的替代方案？ | 现有老系统为 C/S 架构，无法与巴枪对接，需全量改造 |

## 二、需求目标

| 问题 | 你的回答 |
|------|---------|
| 期望达到的结果是什么？ | 收货流程全电子化，支持巴枪扫码、PC 端审核、自动 PIX 记账 |
| 成功指标？ | 收货效率提升 50%，错误率降低至 0.1% 以下，GSP 00201 100% 符合 |
| 优先级？ | P0 紧急（药监检查前必须上线） |
| 期望上线时间？ | 2026-09-30 |

## 三、功能范围

| 问题 | 你的回答 |
|------|---------|
| 功能边界定义（包含什么） | ASN 单收货、扫码验证、上架、PIX 记账、异常处理 |
| 明确排除项（不包含什么） | 不含 ASN 预到货（独立模块）、不含上架策略配置（用 WMOS 原生） |
| 依赖系统有哪些？ | WMOS（核心）、PKMS（质量状态）、TMS（在途状态） |
| 是否有接口对接需求？ | 与 WMOS PIX 事务接口对接（入库收货场景） |

## 四、GSP 合规要求

| 条款编号 | 缺陷等级 | 是否涉及 | 备注 |
|---------|---------|---------|------|
| **00201** 药品追溯系统 | 严重 | ✅ 是 | 必须支持批次+序列号全流程追溯 |
| **05805** 计算机系统 | 严重 | ✅ 是 | 所有功能均涉及 |
| *05701 全过程计算机系统 | 主要 | ✅ 是 | 所有功能均涉及 |
| *05901 数据授权 | 主要 | ✅ 是 | 需按角色分权（仓管/复核/质检） |
| 04001-04003 更改留痕 | 主要 | ✅ 是 | 所有操作需审计追踪 |
| 第八十五条 效期锁定 | 一般 | ✅ 是 | 过期拦截、近效期预警 |
| 第八十七条 质量锁定 | 一般 | ✅ 是 | 不合格品自动冻结 |

## 五、特殊要求

- 手持终端交互：✅ 批量扫描（一次扫多个 SKU）
- 性能要求：单次收货扫码响应 < 1s，并发收货 50 单/分钟
- 其他约束：必须支持离线模式（仓库网络不稳定）
