# Figma 设计文档规范

> 适用于阶段 4（系统功能方案）
> 输出物：Figma 设计文档（不是可执行代码）

## 何时需要

| 需求类型 | 是否需要 |
|---|---|
| 简单需求（加字段/调规则） | ❌ 跳过 |
| 中等需求（新增功能/页面改造） | ✅ 需要 |
| 复杂需求（跨模块/多角色/算法） | ✅ 需要 |

## 输出规范

### 文档结构

```markdown
# Figma 设计文档：{功能名称}

## 全局设计规范
| 项目 | 规范值 | 从UI规范提取 |

## 页面清单
| 页面ID | 页面名称 | 类型 | 尺寸 | 说明 |

## 页面 PC1：{页面名称}
### 布局结构
[ASCII图，标注尺寸/间距/颜色]
### 组件清单
| 组件类型 | 位置 | 规格 | 样式 | 交互 | 数据来源 |

### 状态标签规范（如有状态列）
| 状态码 | 状态名称 | 标签颜色 | 标签样式 |

### 交互流程
1. 页面加载 → ...
2. 用户操作 → ...

### 状态标注
| 状态 | 视觉表现 |

## 页面跳转关系图
[ASCII流程图]

## 响应式适配说明
```

## UI 规范遵循（强制）

按目标端严格遵循对应 UI 规范：

| 终端类型 | 规范文件 | 关键参数 |
|---|---|---|
| PC 端 | `{项目}/PC端UI设计规范_速查版.md` | 1920px 基准、8px 栅格、Element Plus |
| Pad 端 | `{项目}/Pad端UI设计规范_速查版.md` | 1920px 基准、8px 栅格、uView |
| 移动端 | `{项目}/移动端UI设计规范_速查版.md` | 1080×1920px、4px 栅格、uView |

## 设计规范检查清单

生成 Figma 文档前逐项核对：
- [ ] 字体：中文字体、数字字体、字号层级
- [ ] 色彩：主题色、辅助色、状态色、中性色
- [ ] 按钮：尺寸、圆角、交互状态
- [ ] 图标：尺寸、圆角、风格
- [ ] 弹窗：尺寸、类型、布局
- [ ] 栅格：基准宽度、间距系统、栅格列数

## 命名

`Figma设计文档_{功能名}_{端标识}.md`

存放：`{项目设计工场路径}/{项目名}/`（如 `~/design/{项目名}/` 或 `/workspace/designs/{项目名}/`）

## 与 PRD 的关系

- Figma 设计文档：设计规格，供用户确认
- PRD：产品需求，含功能、规则、权限、异常

两者**不重复**：Figma 关注"长什么样"，PRD 关注"做什么、为什么"。
