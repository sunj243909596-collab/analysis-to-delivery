# 业务需求文档(BRD):客户订单管理

> 项目:SaaS 后台 — 客户订单管理模块
> 版本:v1.0
> 日期:2026-06-22
> 状态:✅ 已确认(待阶段 3-9 展开)

## 一、项目概述

### 1.1 业务背景

B2B 中小企业当前用 Excel + 微信群管理订单,存在以下问题:
- 效率低:单订单处理平均 30 分钟
- 易出错:手抄错误率约 5%
- 难追溯:无系统化审计,无法满足财务对账要求
- 缺统计:销售数据靠人工汇总

### 1.2 项目目标

- ✅ 订单全流程在线化(下单 → 支付 → 发货 → 完成)
- ✅ 处理效率提升 50%(30 分钟 → 15 分钟)
- ✅ 错误率降低至 0.5% 以下
- ✅ 多租户数据隔离 100%
- ✅ 数据看板 T+1 准确率 100%

### 1.3 项目范围

**包含**:
- 客户/商品/订单基础管理
- 订单全流程(创建→支付→发货→完成)
- 支付集成(支付宝/微信/Stripe)
- 物流跟踪(顺丰 API)
- 数据看板
- 多租户隔离

**不包含**:
- 财务记账(用现有 ERP)
- 仓储管理(用现有 WMS)
- 移动端 APP / 小程序(v1 仅响应式 Web)

## 二、角色与职责

| 角色 | 职责 | 权限 |
|---|---|---|
| 客户(下单人) | 下单/查询/支付/申请退款 | 看自己的订单 |
| 销售(跟单员) | 审核订单/确认收款/安排发货 | 看所辖客户订单 |
| 仓管 | 拣货/发货/更新物流 | 改订单状态 |
| 管理员 | 客户管理/权限分配 | 全部 |
| 数据看板 | 看销售统计 | 只读 |

## 四、功能模块

- 支付回调
- 发货管理

- 退款流程

- 按商品统计
- 趋势图

## 六、非功能需求

### 6.1 性能
- 单订单创建响应 < 500ms
- 并发下单能力:100 单/秒
- 数据看板 T+1 加载 < 3s

### 6.2 可靠性
- 支付回调幂等性(防止重复回调)
- 多租户数据严格隔离
- 审计日志完整

### 6.3 安全
- 多租户数据隔离(应用层 + DB 层)
- HTTPS 强制
- 密码 bcrypt 加密
- 操作日志完整

### 6.4 可维护性
- 严格分层(Controller / Service / Repository / Entity)
- TypeScript 严格模式
- 单元测试覆盖率 ≥ 80%

## 七、风险与约束

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 支付通道不稳定 | 订单卡在"待支付" | 设计降级(允许手工确认支付) |
| 物流 API 限流 | 物流更新延迟 | 异步队列 + 重试 |
| 多租户数据泄露 | 严重 | 所有查询强制带 `tenant_id`(代码审查重点) |
| 并发超卖 | 库存负数 | DB 悲观锁 + 乐观锁组合 |
