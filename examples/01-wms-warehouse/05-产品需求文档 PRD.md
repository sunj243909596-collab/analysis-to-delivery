# 产品需求文档（PRD）：医药物流 WMS 收货管理

> 版本：v0.1
> 日期：2026-07-08
> 状态：✅ 已确认

## 一、产品概述

### 1.1 背景
医药仓库到货收货环节,从纸质单据+手工录入,升级为巴枪扫码+PC审核全电子化流程,解决 GSP 00201 追溯合规 + 效率/准确率问题。详见 `01-业务需求文档 BRD.md` §一。

### 1.2 目标
- 收货效率提升 50%（30 分钟 → 15 分钟 / 单 ASN）
- 错误率降至 0.1% 以下
- GSP 00201/05805/05701/05901/04001-04003 + 第八十五/八十七条 全合规
- PIX 记账零失败（事务保证）

### 1.3 范围
**包含**：ASN 扫码收货、SKU+批号+序列号+效期校验、上架、PIX 记账、PC 端审核、异常处理（短收/拒收/超收）。
**不包含**：ASN 预到货（独立模块）、上架策略配置（沿用 WMOS 原生）。

## 二、用户故事

| 角色 | 场景 | 需求 | 优先级 |
|---|---|---|---|
| 仓管员 | 到货后扫码收货 | 扫 ASN → 显示待收清单 → 逐项扫 SKU/批号/序列号 → 系统实时校验 → 提交收货 → 离线模式可用 | P0 |
| 复核员 | PC 端审核收货单 | 收到审核待办 → 比对 ASN 应收 vs 实收 → 确认/驳回 → 处理异常任务 | P0 |
| 质检员 | 处理异常 | 收到短收/拒收/质量异常待办 → 录入原因 → 决定补货/退货/让步接收 | P0 |
| 系统管理员 | 配置权限/参数 | 4 角色权限分配;PIX 事务版本;效期阈值(默认 90 天);设备绑定 | P1 |

## 三、功能需求

### 3.1 巴枪扫码收货

#### 3.1.1 功能说明
仓管员在巴枪端扫 ASN 单号 → 系统加载待收清单 → 仓管员依次扫 SKU/批号/序列号 → 系统实时校验 → 提交收货触发 PIX 记账。

#### 3.1.2 业务流程
详见 BRD §三.3.1 泳道图 + `业务流程图-收货流程.txt`。

#### 3.1.3 界面设计
（巴枪端：参考 uni-app 模板,4 步骤向导：扫 ASN → 扫 SKU+批号 → 扫序列号 → 提交。屏幕 4 寸,大按钮 ≥ 60px）

#### 3.1.4 字段定义

| 字段名 | 类型 | 必填 | 校验 | 数据来源 | 备注 |
|---|---|---|---|---|---|
| TC_ASN_ID | VARCHAR2(20) | 是 | 非空 + ASN 表存在 | ASN | ❌ 严禁用 ASN_NBR |
| SKU | VARCHAR2(30) | 是 | 非空 + ITEM 表存在 | ASN_DETAIL | |
| BATCH_NBR | VARCHAR2(20) | 是 | 非空 + 与 ASN_DETAIL 匹配 | ASN_DETAIL | ❌ 严禁用 LOT_ID |
| SN | VARCHAR2(40) | 条件 | SKU 序列号管控=是时必填 | SN_MASTER | ❌ 严禁用 EPC/GTIN 替代 |
| EXP_DATE | DATE | 是 | > SYSDATE + 90 天 | ITEM_LOT | 效期阈值可配 |
| QTY | NUMBER | 是 | > 0 且 ≤ ASN_DETAIL 剩余应收 | 输入 | 整数,严禁 1/3 必须 1.0/3 |

#### 3.1.5 业务规则
- 效期过期 → 拒收,直接走质检异常
- 质量状态不合格（来自 PKMS）→ 拒收
- 序列号重复 → 拦截 + 提示已存在位置
- 批次与 ASN 不符 → 拦截 + 提示
- 超收不允许（系统硬拦截）
- 短收 → 提交短收单,触发补货流程
- 离线模式：本地存到 APP_RCV_TASK_OFFLINE,网络恢复后按时间戳回放

#### 3.1.6 异常处理

| 异常 | 提示 | 处理 |
|---|---|---|
| ASN 不存在 | "ASN 单号错误,请确认" | 重新扫 |
| 批次错误 | "批次 {x} 与 ASN 不匹配,应收 {y}" | 重新扫 |
| 序列号重复 | "序列号 {x} 已在 {位置} 收货" | 复核/联系 QA |
| 效期过期 | "效期 {date} 已过期 < 90 天阈值" | 走质检拒收 |
| 离线 | "当前离线,扫码暂存本地" | 联网后自动同步 |

#### 3.1.7 权限控制
- 仓管员：可创建/提交收货、可登记异常
- 复核员：可审核/驳回、不可创建收货
- 质检员：可处理异常、不可创建收货
- 系统管理员：只读 + 配置权限

### 3.2 PC 端审核收货单

#### 3.2.1 功能说明
复核员 PC 端登录 → 收到待办 → 比对 ASN 应收 vs 实收 → 确认/驳回。
- 触发 PIX：RCV_VERIFY 事务（状态 30→40）

#### 3.2.2 业务规则
- 应收 vs 实收一致 → 通过
- 实收 < 应收 → 短收,需选择"补货/让步接收"
- 实收 > 应收 → 硬拦截,需仓管员先撤单重做

#### 3.2.3 字段定义

| 字段名 | 类型 | 必填 | 校验 | 数据来源 |
|---|---|---|---|---|
| ASN_ID | NUMBER | 是 | ASN 表存在 | ASN |
| RECEIVED_QTY | NUMBER | - | > 0 | APP_RCV_RECORD 聚合 |
| EXPECTED_QTY | NUMBER | - | > 0 | ASN_DETAIL 聚合 |
| VERIFY_STATUS | VARCHAR2(10) | 是 | 枚举(APPROVE/REJECT) | 输入 |
| REMARK | VARCHAR2(500) | 否 | - | 输入 |

#### 3.2.4 权限控制
- 复核员：可审核/驳回、可看明细
- 仓管员：只看自己提交的
- 质检员：只看异常任务

## 四、非功能需求

### 4.1 性能
- 单次扫码响应 < 1s（P95）
- 并发收货：50 单/分钟（≥5 个仓管员同时工作）
- PIX 记账延迟 < 3s（P95）

### 4.2 安全
- 巴枪设备绑定（设备 ID + 仓管员 ID 双重认证）
- 操作日志完整记录（5 件套 + 业务日志）
- 敏感数据脱敏（供应商手机号 `138****1234`）

### 4.3 兼容性
- 巴枪端：Android 10+ / iOS 14+
- PC 端：Chrome 100+ / Edge 100+
- 后端：Java 11 / Spring Boot 2.7 / Oracle 19c
