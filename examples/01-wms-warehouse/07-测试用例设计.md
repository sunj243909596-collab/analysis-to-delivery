# 测试用例设计：医药物流 WMS 收货管理

> 版本：v0.1
> 日期：2026-07-10
> 状态：✅ 已确认

## 一、测试范围

本测试用例覆盖医药物流 WMS 收货管理的全部功能,包括：
- 巴枪扫码收货主流程
- PC 端审核
- 异常处理（短收/拒收/超收/序列号异常/批次错误）
- GSP 合规校验（00201/05805/05701/05901/04001-04003/第八十五/八十七条）
- 性能与安全

测试范围以 `01-业务需求文档 BRD.md` §四 功能模块为准,涵盖 8.1-8.4 巴枪端 + 9.1-9.3 PC 端共 7 个核心模块。

## 二、测试策略

| 测试类型 | 范围 | 工具 |
|---|---|---|
| 单元测试 | Service / Repository 层(收货核心业务) | JUnit 5 + Mockito |
| 集成测试 | Controller → Service → DB 全链路 + PIX 调用 | Spring Boot Test + Testcontainers (Oracle) |
| E2E 测试 | 巴枪 → 后端 → PC → WMOS 全链路 | uni-app 自动化 + Playwright |
| 性能测试 | 50 单/分钟并发 + 扫码响应 < 1s | JMeter |
| 安全测试 | 设备绑定 + 权限 + SQL 注入 + 敏感数据脱敏 | OWASP ZAP + 手工 |
